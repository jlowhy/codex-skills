# Evidence Standard

Every substantive claim needs repo-relative evidence: file references, commands, configs, observed behavior, or runtime output.

Distinguish observed fact from inference. Treat docs as secondary when executable config or code contradicts them.

Prefer the documented root command surface before lower-level package commands. Use lower-level commands only to verify delegation or diagnose a mismatch.

Mark relevant areas that could not be inspected or executed as `[not covered]`, with the reason and impact on confidence.
